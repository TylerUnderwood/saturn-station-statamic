Intro

This is where we build the core template, the foundation upon which all the other markup lays. Any upates should be minimal and not subect to changes. The template has a strict order of nesting. First inside the body element, should be the template div. Then inside of the template div should be, in order, header, main, and footer elements. Here is what the template must contain.

  • HTML <html>
  • Body <body id="body">
  • Site <div id="site" class="template-class">
  • Header <header id="header">
  • Main <main id="main">
  • Footer <footer id="footer">
  • Section <section id="section_one" class="section">

HTML

The html tag has the root text size which I've set it to be 16px below a screen width of 480px and up to 20px at a screen width of 1600px. This is a fluid width, so anywhere between 480px and 1600px will be dynamicly calculated for a smooth transition.

Body

The body element cuts off horizontal overflow, sets the defaut text styles and the background color for the whole site. I've set the default font size to 1rem to ensure all text folows the fluid font at the root. The text color and background color are set to a css var so you can easily change it for themes. So, if you want a new theme just make a class and add it to the body.

Site

The #site element can be used to hook in Vue or other frameworks. This will, at the least, fill the viewport so that the site is never smaller than the browser window. The site element also gives the template structure, almost all cases a flex column is enough, so I have set that as the default. If you want it to change just add a class.

Header

The header comes first, but should always be on top.

Main

The main is in the middle, and it should fill the space between, but be below both the header and footer.

Footer

The footer is at the bottom, below the header but above the main.

The Flow

<html>
  <head>Meta or whatevs</head>
  <body id="body">
    <div id="site" class="template-class">

      <header id="header">Header stuff</header>

      <main id="main">

        This can be whatever, but it is usualy like this

        <section id="section_one">Section stuff</section>

        <section id="section_two">Section stuff</section>

        <section id="section_three">Section stuff</section>
        
      </main>

      <footer id="footer">Footer stuff</footer>

    </div>
  </body>
</html>
